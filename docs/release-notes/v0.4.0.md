# AI Chat Shell Exec v0.4.0

Protocol boundary stabilization release.

## Highlights

- Server health now reports release version, server protocol version, helper protocol version, helper protocol name, and tmux workspace defaults.
- The extension requires server protocol `2` and helper protocol `1`.
- Old foreground servers are rejected with an actionable restart message instead of failing later during command execution.
- Floating-panel, popup, and doctor checks now show server release/protocol/helper details alongside `ForAI` readiness.

## Upgrade Notes

1. Download and unzip `ai-chat-shell-exec-extension-v0.4.0-source.zip`.
2. Reload the unpacked extension from the matching `extension/` directory.
3. Stop any older foreground shell server.
4. Start the current server with `./scripts/start_shell_server.sh`.
5. Use the popup or floating-panel `Check` action and confirm it reports server protocol `2`, helper `1`, and `ForAI` readiness.
