# AI Chat Shell Exec v0.9.1

This patch fixes automatic composer delivery reclaiming the input after a user intentionally deletes or replaces inserted content.

## What changed

- Every helper result or agent prompt may be written into the composer automatically at most once.
- Deleting or replacing inserted shell/board output cancels that delivery and every result already queued behind it. The extension does not reinsert the content and does not mark it presented.
- A genuinely new rendered helper can still ask the backend to recover an unpresented result; user cancellation is not command-execution dedup.
- Deleting, replacing, or navigating away from an inserted agent prompt cancels its composer delivery. If local hub acknowledgement is temporarily unavailable, only that acknowledgement retries.
- File, agent-message, roster, task-status, and other helper responses no longer re-execute the same rendered helper when composer sending cannot be confirmed.
- A pre-execution failure remains scanner-retryable only when the extension never mutated the composer.

## Validation

- Added regression coverage reproducing deliberate deletion after a failed send and proving there is no second composer write or send attempt.
- Added coverage for cancelling already-queued helper results, retrying only an agent cancellation acknowledgement, and keeping side-effecting non-shell helpers single-execution after a composer write.
- Added a real Chromium E2E that blocks sending, waits for actual shell output in the composer, deliberately deletes it, waits across multiple retry intervals, proves it stays deleted and the command ran once, then verifies a new helper still delivers normally.
- The complete static, unit, integration, tmux, and Chromium E2E suite passed before release.
