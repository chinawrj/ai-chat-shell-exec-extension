# AI Chat Shell Exec v0.3.4

ForAI workspace reliability release.

## Highlights

- New `ForAI` `host` and `board` tmux windows start in the project root by default.
- Set `AI_CHAT_SHELL_FORAI_CWD=/path/to/workspace` before starting the local server to choose a different default cwd.
- Shell helpers no longer support a target line; every line between `ai-helper-shell-start` and `ai-helper-shell-end` is command text, so heredocs and `cat <<EOF` file creation work in one helper block. Legacy shell target fields are ignored and commands run in `ForAI:host`.
- The floating panel and popup now include reset actions that recreate only the default `ForAI` tmux session, including when that session is already missing.
- Startup, `Check`, popup, health, and `scripts/doctor.sh` diagnostics report host, board, cwd, and server protocol state.
- The Chrome extension e2e test now exercises the no-target shell helper path and verifies default execution in `ForAI:host`.

## Install Or Upgrade

1. Download and unzip `ai-chat-shell-exec-extension-v0.3.4-source.zip`.
2. Open `chrome://extensions`, enable Developer mode, and load the extracted project root or `extension/`.
3. Restart the local shell server from the extracted project root.

   On macOS:

   ```sh
   ./scripts/install_shell_server_agent.sh
   ```

   On Ubuntu, or for a temporary foreground server:

   ```sh
   ./scripts/start_shell_server.sh
   ```

4. Reload enabled chat tabs.
5. Use the floating panel's `Check` button and confirm it reports `ForAI ready`, host, board, and cwd.
6. Run `./scripts/doctor.sh` if the page reports tmux or server state as unavailable.
