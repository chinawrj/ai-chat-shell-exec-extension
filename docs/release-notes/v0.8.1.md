# AI Chat Shell Exec v0.8.1

This patch release hardens manual Force run behavior.

## Changes

- Fixes a case where clicking `Force run` while another helper was active could be dropped after the old bounded retry window expired.
- Keeps the forced run request pending until the current helper clears, then executes the latest detected helper with force metadata.
- Accepts force metadata consistently as either `callMeta.force=true` or top-level `force=true` in background message handlers.
- Marks server ledger entries as `failed` when command execution throws after a claim, avoiding stale `running` locks.
