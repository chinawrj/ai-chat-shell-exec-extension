# AI Chat Shell Exec v0.3.5

State directory hardening release.

## Highlights

- The shell server now preflights its runtime state directory before listening and automatically repairs safe state-path conflicts by moving the bad path aside with a `.broken-*` suffix.
- `AI_CHAT_SHELL_STATE_DIR=/path/to/state` can move runtime state when needed.
- `scripts/start_shell_server.sh` creates and validates the state directory before launching the server.
- macOS LaunchAgent auto-start has been removed. Use the foreground server start script; the old `./scripts/install_shell_server_agent.sh` command now removes legacy LaunchAgents and then starts the same foreground server.
- `/health` and `scripts/doctor.sh` report `stateOk`, `stateDir`, any state error, and repair actions.
- The state directory is treated as rebuildable runtime data. Removing it should not be required, but if it is removed the server recreates the required layout.

## Install Or Upgrade

1. Download and unzip `ai-chat-shell-exec-extension-v0.3.5-source.zip`.
2. Open `chrome://extensions`, enable Developer mode, and load the extracted project root or `extension/`.
3. Restart the local shell server from the extracted project root.

   ```sh
   ./scripts/start_shell_server.sh
   ```

4. Reload enabled chat tabs.
5. Run `./scripts/doctor.sh` and confirm state, server, tmux, host, board, and cwd checks pass.
