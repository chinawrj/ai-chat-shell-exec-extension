# AI Chat Shell Exec v0.8.4

This patch release improves master/slave recovery and makes roster information safer to act on.

## Changes

- Treats saved agent profiles as panel defaults only; a new tab must be explicitly saved before it polls or consumes local agent messages.
- Persists pending sent-but-unacked agent deliveries so reloads retry only the local ack instead of reinserting or resending a task.
- Marks missing tmux-ai panes as stale in async roster results and reports stale tmux-ai slaves as `canReceiveTask=false`.
- Adds clearer recovery hints for stale tmux-ai panes and repeated send-button detection failures.
- Expands Chrome E2E to cover real slave panel registration, real master `agent-message` helper delegation, and the real tmux-ai Refresh dropdown path.
