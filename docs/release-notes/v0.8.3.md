# AI Chat Shell Exec v0.8.3

This patch release improves local master/slave reliability and floating-panel diagnostics.

## Changes

- Treats browser-tab slaves as ready Agent Check targets; tmux-ai is now optional for browser-only master/slave workflows.
- Marks tmux-ai slaves as stale when their registered tmux pane is no longer visible.
- Keeps web agents heartbeating while a helper call is running, so long-running work does not make the agent disappear from the roster.
- Shows repeated agent polling failures in the floating panel with an Agent Check recovery hint.
- Renames the floating-panel health button to `Server Check` and the agent diagnostics button to `Agent Check`.
- Keeps browser-slave reply templates quoted so placeholder reply helpers are not executed accidentally.
- Adds Chrome E2E coverage for browser-only Agent Check readiness, roster helpers, task-status helpers, and slave reply template safety.
