# AI Chat Shell Exec v0.3.0

Board helper release.

## Highlights

- Adds `ai-helper-board-start` / `ai-helper-board-end` blocks for asking the human helper for board output.
- Sends board commands to `AI_CHAT_SHELL_BOARD_TARGET` when set, otherwise to the unique tmux window named `board`.
- Probes the current board prompt before every board command; if no prompt is found, the command is not sent.
- Captures board output from tmux `pipe-pane` logs and stops when the probed prompt returns.
- Renders common terminal redraw sequences such as backspace, carriage return, line clear, and cursor movement so returned output matches the visible tmux pane more closely.
- Updates README and AI-facing instruction samples to require fenced `text` code blocks for shell, board, and file helpers.

## Board Helper Format

```text
ai-helper-board-start
version
ai-helper-board-end
```

The board helper body must be exactly one non-empty board command line. It does not include a tmux target or cwd.

## Upgrade Notes

1. Download and unzip `ai-chat-shell-exec-extension-v0.3.0-source.zip`.
2. Open `chrome://extensions`, enable Developer mode, and load the extracted project root or `extension/`.
3. From the extracted project root, restart the local shell server.

   On macOS:

   ```sh
   ./scripts/install_shell_server_agent.sh
   ```

   On Ubuntu, or for a temporary foreground server:

   ```sh
   ./scripts/start_shell_server.sh
   ```

4. Reload enabled chat tabs and confirm the floating panel shows `Shell tool ready v0.6.31`.
5. If using board helpers, ensure tmux has exactly one window named `board`, or set `AI_CHAT_SHELL_BOARD_TARGET` to a pane id such as `%0`.
6. Update AI instructions from `README.md` or `docs/AI_INSTRUCTIONS.md`; the examples should say to use fenced `text` code blocks and no prose.
