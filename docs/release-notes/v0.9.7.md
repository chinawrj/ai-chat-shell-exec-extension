# AI Chat Shell Exec v0.9.7

This release restores the composer send actuator to the known v0.8.9 behavior while retaining the execution, recovery, and delivery fixes added afterward.

## What changed

- Restored `clickSendWhenReady`, `trySubmitForm`, `tryKeyboardSubmit`, `waitForSubmitted`, and `findSendButton` byte-for-byte from v0.8.9.
- Removed the later cumulative button/form/keyboard actuation budget and the later send-button association filters.
- Added an exact full-SHA-256 source-lock regression test for all five v0.8.9 functions.
- Kept lifecycle and exact-composer ownership enforcement outside those functions. Each invocation still gets the original bounded 80-round send-readiness loop; if ownership changes while it is running, stale side effects are blocked, and a persistent delivery retry cannot automatically restart another 80-round loop every two seconds within the same page lifecycle.
- Runs the original short waits through a bounded background message clock so hidden-tab throttling cannot stretch the first 20 readiness rounds into apparent non-delivery.
- Preserves a rendered helper's handled proof only while an exact pending result crosses a same-tab route change, preventing route-triggered duplicate file/helper execution without suppressing a genuinely new SPA conversation.
- Captures stale actuator events on both the document and the original composer subtree, continuously rejects an editor that is no longer the current candidate, rejects detached/hidden editors and trusted user mutations, aborts the old loop at its next wait boundary after ownership loss, and allows one bounded handoff only when the replacement composer still contains the exact plugin-owned text.
- Carries `queued` results that have already received a backend response across the same-tab route transition, so temporary composer unavailability cannot discard the result or make a file/board helper eligible for a second backend request.

## What remains

- The shell server remains the final and sole duplicate authority.
- Ctrl+C completion, per-pane serialization, refresh recovery, canonical execution IDs, and local-only duplicate diagnostics remain unchanged.
- Durable one-write pending delivery, SPA route recovery, composer redraw recovery, intentional-deletion cancellation, and user-draft protection remain in place.

## Validation

- Focused content, route, and agent delivery tests.
- Full standalone test suite.
- Real Chromium unpacked-extension E2E, including submission while the helper page is an explicitly hidden tab.
- Independent final review by three agents.
