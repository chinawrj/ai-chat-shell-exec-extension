# AI Chat Shell Exec v0.8.8

This patch fixes shell helpers that are interrupted with Ctrl+C inside tmux.

## What changed

- An actually started shell command now returns immediately after Ctrl+C with exit code 130 and explicit interruption metadata.
- Ctrl+C is no longer reported as a shell state timeout.
- Interrupted commands count as executed history. An identical command on the same resolved tmux pane and actual cwd can therefore be returned as a server-authoritative duplicate; use **Force run** to intentionally run it again.
- A command interrupted before the server-controlled executed marker is written remains retryable.
- The tmux runner no longer backgrounds the command, preventing an orphaned child from keeping the server waiting.

## Validation

- Added an isolated real-tmux test that sends an actual `C-c`, requires a prompt server response, verifies exit code 130 and interruption metadata, and confirms that the next identical request is deduplicated.
- Added unit coverage for runner signal traps, persistent interrupted-execution ledger state, duplicate response diagnostics, and rendered `shell-output` metadata.
- Passed the complete automated test suite and final multi-agent review before release.
