# AI Chat Shell Exec v0.9.6

This release closes the final route-only and cumulative-fallback gaps found by the post-v0.9.5 team audit.

## What changed

- Pending helper retries and agent polling now detect same-tab URL changes before loading route-scoped storage. A `pushState`/`replaceState` with no DOM mutation can no longer strand exact plugin-owned text in the composer.
- Every button, form, or keyboard send side effect revalidates the fresh current visible composer. A connected but hidden old editor is not send authority.
- Saved localized Send controls must still be structurally or geometrically associated with the current composer.
- Helper and agent pending deliveries persist one cumulative five-actuation envelope across retry calls and reload recovery: at most three button clicks, one form submit, and one keyboard submit. A no-op button cannot consume the capacity reserved for the other fallbacks.

## Safety

- The extension writes plugin output only once.
- Different user text is never overwritten, adopted, or sent.
- Backend execution is never repeated to recover a composer/send failure.
- Exhausting the bounded automatic actuation envelope leaves the exact plugin-owned text available for manual Send instead of continuing indefinitely.

## Validation

- Route-only helper and agent tests change the URL without manual lifecycle refresh or DOM mutation and prove send-only recovery.
- Hidden connected composer and stale localized binding tests require zero send side effects.
- Cross-call actuation tests prove the cumulative budget cannot reset on retry.
- The full standalone suite and unpacked-extension Chromium E2E are required before release.
