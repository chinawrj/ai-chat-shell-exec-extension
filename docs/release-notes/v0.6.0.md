# AI Chat Shell Exec v0.6.0

Local multi-agent tabs release.

## Highlights

- Register enabled chat tabs as local `master` or `slave` agents from the floating panel.
- Delegate tasks between tabs with `ai-helper-agent-message-start` / `ai-helper-agent-message-end` helper blocks routed through the local WebSocket hub.
- Run shell helpers from agent tabs in isolated tmux workspaces such as `ForAI-slave-a:host`; non-agent tabs still use the default `ForAI:host`.
- Keep active agents online with poll-based heartbeats and automatically re-register tabs if the local server roster is lost after a restart.
- Prevent duplicate task insertion when an agent message is waiting for the chat send button.
- Show clearer `Save` and `Roster` controls, suggested agent ids, and pending message counts in the floating panel.
- Require both sender and recipient agents to be registered before the hub accepts a message.
- Add master/slave instruction templates in `docs/AI_INSTRUCTIONS.md` and a local manual test path in `tests/manual/tmux-test-page.html`.

## Upgrade Notes

1. Download and unzip `ai-chat-shell-exec-extension-v0.6.0-source.zip`.
2. Reload the unpacked extension from the matching `extension/` directory.
3. Stop any older foreground shell server.
4. Start the current server with `./scripts/start_shell_server.sh`.
5. Refresh enabled chat tabs so the content script version matches the background worker.
6. Use the popup or floating-panel `Check` action and confirm it reports server protocol `3`, helper `1`, and `ForAI` readiness.
7. To use local multi-agent tabs, configure one tab as `master`, configure one or more tabs as `slave`, click `Save` on each, then click `Roster` to confirm the registered agents.
