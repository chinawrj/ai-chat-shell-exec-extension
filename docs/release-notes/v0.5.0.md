# AI Chat Shell Exec v0.5.0

Local visual tmux adapter release.

## Highlights

- Adds visual tmux surface discovery for macOS windows, defaulting to Terminal.app and Ghostty.
- The server now reports visual protocol metadata and supported local visual tmux apps from `/health`.
- The shell server protocol is now `3`; helper protocol remains `1`.
- The extension background forwards supported visual discovery/run requests only after the same server health and stale-server checks used by shell, file, board, and tmux flows.
- The existing OCR execution path remains single-line and uses visible tmux window/status markers plus OCR pagination for output reconstruction.
- Visual command execution is constrained to Terminal.app/Ghostty local tmux surfaces in this release.

## Upgrade Notes

1. Download and unzip `ai-chat-shell-exec-extension-v0.5.0-source.zip`.
2. Reload the unpacked extension from the matching `extension/` directory.
3. Stop any older foreground shell server.
4. Start the current server with `./scripts/start_shell_server.sh`.
5. Use the popup or floating-panel `Check` action and confirm it reports server protocol `3`, helper `1`, and `ForAI` readiness.
6. Build the macOS vision helper with `./scripts/build_macos_vision_helper.sh` before using local visual tmux surfaces.
