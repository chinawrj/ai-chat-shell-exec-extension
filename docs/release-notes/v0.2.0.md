# AI Chat Shell Exec v0.2.0

tmux-backed release for routing command requests into an existing terminal pane.

## Highlights

- Commands now require an explicit tmux target such as `%24`, `espcam:0.0`, or a unique window name like `build`.
- The server no longer spawns a fresh `/bin/zsh -lc` for every request. It sends the command into the selected tmux pane and captures output between completion markers.
- The popup lists available tmux panes, including pane id, address, window name, current command, current path, and active state.
- Missing or invalid targets are rejected with a target list and a JSON example.
- AI instructions now use neutral terminal-output wording and keep requests minimal: `{"target":"%24","cmd":"pwd"}`.
- The content script avoids stale-history triggers by narrowing scanning to the active chat feed when available.
- Duplicate protection now includes `target`, so identical commands in different panes are not conflated.

## Diagnostics And Testing

- Adds `scripts/doctor.sh` for install and runtime checks.
- Adds a local HTTPS tmux manual test page.
- Adds a helper for launching an isolated Chromium-family profile with the unpacked extension.
- Adds tmux helper coverage, manifest consistency checks, popup config coverage, and WebSocket frame parser tests.

## Install Or Upgrade

1. Download and unzip `ai-chat-shell-exec-extension-v0.2.0-source.zip`.
2. Open `chrome://extensions`, enable Developer mode, and load the extracted project root or `extension/`.
3. From the extracted project root, run:

   ```sh
   ./scripts/install_shell_server_agent.sh
   ```

4. Make sure tmux is running and that the target pane is a shell prompt ready to receive commands.
5. Use the popup tmux target list to pick a target, then ask the AI to request commands using `shell-call` JSON with `target` and `cmd`.

## Compatibility Notes

- Plain command-only `shell-call` blocks are rejected. This is intentional to avoid sending commands to the wrong tmux pane.
- Shell-like `bash`, `sh`, `zsh`, and `shell` blocks still require a `# local-shell` marker or explicit tool-language context.
- First version uses completion snapshots, not live streaming output.

## Security

This release still enables local command execution requested from AI chat pages. Use it only on machines you control, keep enabled sites narrow, and review commands carefully when working with sensitive projects.
