# AI Chat Shell Exec v0.8.5

## Changes

- Changed ai-helper shell timeout semantics to pid/status-aware waiting: the configured shell state timeout is no longer a command runtime limit, and the server keeps waiting while the tmux runner process is still alive.
- Added per-run pid and completion-status files for tmux shell runs so missing completion markers can report explicit `timeoutReason`, `processKnown`, and `processAlive` metadata.
- Removed the extension-side WebSocket runtime watchdog for shell `run` messages while keeping watchdogs for shorter non-shell operations.
- Added shell-output metadata for timeout/process-state diagnostics, and relabeled the popup setting as `State timeout ms`.
