# AI Chat Shell Exec v0.8.0

This release makes local multi-agent workflows usable from the AI master's point of view.

## Changes

- Adds `ai-helper-agent-roster-start` / `ai-helper-agent-roster-end` so a master AI can query online slave agents directly.
- Adds `ai-helper-agent-task-status-start` / `ai-helper-agent-task-status-end` so a master AI can check delegated tasks by `message-id` or `task-id`.
- Extends roster metadata with slave surface, reply mode, pending count, `canReceiveTask`, capabilities, `lastSeenAgeMs`, and tmux target names.
- Adds AI-oriented recovery hints for missing recipients, duplicate messages, stale replies, and unavailable tmux-ai panes.
- Updates README and AI-facing instructions to use a roster -> delegate -> status workflow.
- Bumps helper protocol compatibility to `2`; restart `./scripts/start_shell_server.sh` after updating the extension.
