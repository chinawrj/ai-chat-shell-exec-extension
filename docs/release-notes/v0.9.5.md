# AI Chat Shell Exec v0.9.5

This follow-up incorporates the final team audit of the v0.9.4 emergency fix and strengthens its real-browser proof.

## What changed

- Send-button retries are capped at three total clicks: the original two attempts plus one delayed-readiness fallback. Every click still requires complete exact composer ownership.
- A saved Send selector can no longer win merely because heuristics found nothing. Strict localized send labels remain eligible without weakening stale-selector safety.
- Controlled contenteditable fallback restores a real full-editor selection before dispatching `beforeinput`.
- Same-tab route handoff now also preserves submitted/presented deliveries waiting only for their server receipt, plus local presentation tombstones.
- Route storage migration keeps the old snapshot unless writing the new snapshot succeeds.

## Validation

- Helper and agent exact-text route handoff tests passed.
- Backend-failure and rejected-feedback tests prove one backend attempt, one composer write, and send-only retry.
- Presentation-receipt route recovery retries only the receipt and never submits again.
- React tracked textarea, third-click readiness, localized binding, stale-binding safety, and user-draft zero-side-effect tests passed.
- Real Chromium E2E passed with a composer DOM replacement and SPA URL change occurring synchronously during the plugin input event; exactly one full plugin-owned user message was submitted and the current composer was empty.
