# AI Chat Shell Exec v0.2.1

Compatibility fix for fresh ChatGPT installs.

## Highlights

- `chatgpt.com` is enabled by default, so the floating panel appears immediately on ChatGPT after loading the extension.
- Users who still have the old default enabled-site list are migrated to the new default list.
- Custom enabled-site lists are preserved.
- Adds a regression test to keep the default enabled hosts aligned across background, content, and popup code.

## Install Or Upgrade

1. Download and unzip `ai-chat-shell-exec-extension-v0.2.1-source.zip`.
2. Open `chrome://extensions`, enable Developer mode, and load the extracted project root or `extension/`.
3. From the extracted project root, run:

   ```sh
   ./scripts/install_shell_server_agent.sh
   ```

4. Open ChatGPT or M365 Copilot and confirm the lower-right floating panel appears.
5. Use the popup tmux target list to pick a target, then ask the AI to request commands using `shell-call` JSON with `target` and `cmd`.

## Verification

- Confirmed the floating panel appears on a fresh ChatGPT profile.
- Confirmed ChatGPT can autonomously request a `shell-call`, the extension executes it in tmux target `build`, and `shell-output` is posted back.
- Confirmed the local tmux manual test page still lists targets and posts `shell-output`.
