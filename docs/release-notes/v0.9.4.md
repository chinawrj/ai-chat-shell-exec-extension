# AI Chat Shell Exec v0.9.4

This patch fixes the remaining case where plugin-owned text was visible in the composer but automatic submission stopped.

## What changed

- A same-tab SPA URL transition now carries an already-inserted helper result or agent prompt into a send-only retry. It never re-executes the backend or rewrites the composer.
- The transferred delivery must still own the complete current composer text. Empty or different text is treated as user cancellation and is never sent.
- Controlled input and textarea composers are updated through their native value setter so React-style host state sees the change.
- An enabled send button can be retried after the first two no-op clicks, within a bounded deadline. Explicit localized/custom Send bindings remain usable when no current-page heuristic control is available.
- Backend-failure and rejected-helper output now receive the same durable one-write/send-only delivery fallback as successful helper results.

## Validation before release

- Route-change agent regression: stale pre-route token cannot ack; the replacement composer receives send-only retry and one successful ack.
